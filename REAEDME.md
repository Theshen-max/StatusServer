# StatusServer    

**全局状态监测与服务器调度中心**

## 项目目标
实时监控集群内所有 `ChatServer` 的负载情况，为新登录用户分配合适的服务节点，并颁发全局会话令牌（Token）。

## 技术栈
- **服务契约**: gRPC (定义在 `message.proto`)
- **调度算法**: 基于 Redis ZSet 的动态负载均衡（选取 Score/连接数最低的节点）
- **性能优化**: 使用 `thread_local` 优化 UUID 生成，避免多线程锁竞争

## 核心职责
- **智能节点分配**: 通过查询 Redis 发现当前空闲且在线的 `ChatServer`。
- **存活探测**: 结合 Redis TTL 机制，自动剔除因网络异常或宕机未上报心跳的死节点。
- **鉴权中心**: 颁发临时登录 Token，确保客户端向 `ChatServer` 建立长连接时的身份合法性。
